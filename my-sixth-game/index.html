  • Originality: What is original about your game?
    The game itself is supposed to be a hybrid between chess and battleship. Originally, the player wouldn't be able to see the oher players board so attacks would be more random like in battleship. That design didn't work well for the set up of the analog game iteration, so instead, the die creates the randomness and players move according to the die roll.
  • Technical Merit: What is interesting about your game technology?
    It took me a really long to figure out everything. The hardest part was trying to make the pieces draggable. There were issues with the size of the pieces and where their boundaries ended so you would be moving the wrong piece or too many pieces at the same time. It also implements a random die roll. It picks a random number between 1 and 6 when the die is clicked. Once clicked, it updates the text on the screen so that you know how many spaces you can move. When you make your move to attack the other side, you are able to click the gameboard to add a red square to mark your turn and it doesn't interact with your ability to drag the pieces. I also added a sound effect for the die so that it makes a noise when you roll it. I'm working for the next iteration to create it so that you are playing against the computer, but that was not feasible for this iteration.
  • Prototype Postmortem: What did you learn from this prototype?
    What was the easiest or hardest part of making it?
    I did learn how to make pieces draggable and how to update text as the game goes on. The easiest part was setting up the background and the original placess for the pieces. It was hard at first to get the background to the correct size and also hard to format the die correctly. I struggled a lot trying to make the pieces draggable. Like I said above, there were issues with the piece sizes that I could not see on the screen so the wrong pieces were being dragged every time. It took a long time to figure out why it was doing that. I also struggled with updating the next for the die roll. I wasn't sure where to place the die roll function and then place the text update since I wanted them to happen at the same time.
